# Phase 04 Plan 03: Composition Root + OpenAPI Spec + Smoke Summary

Auto-generated OpenAPI spec (`pkg/docs`) from `swag` annotations in Plans 01–02, composition root
`cmd/gateway/main.go` with errgroup lifecycle, Swagger UI served at `/swagger/`, coverage gate
fixed to exclude `pkg/docs` and now passes at 90.4%, Phase-4 curl smoke check, and README
quickstart. All four make gates are green.

## What Was Built

- **`cmd/gateway/main.go`** — composition root: file-level swag annotations (`@title`, `@version 1.0`,
  `@BasePath /api/v1`, `@host localhost:8080`); `main()` wires `signal.NotifyContext(SIGTERM/SIGINT)`,
  `db.FromEnv → db.Migrate → db.New` (defer pool.Close), `gateway.FromEnv → gateway.NewRouter`,
  `http.Server` (ReadTimeout 5s / WriteTimeout 10s), two-goroutine errgroup (ListenAndServe +
  Shutdown coordination). DSN never logged (ASVS V8 / T-04-07).

- **`pkg/docs/`** — generated by `make swagger` (swag v1.16.4): `docs.go` (registers spec on
  `init()` via `swag.Register`), `swagger.json` and `swagger.yaml` with both `/gpus` and
  `/gpus/{id}/telemetry` paths documented. Committed, not gitignored (per plan spec).

- **`internal/gateway/swagger_test.go`** — `TestSwaggerUI` imports `_ pkg/docs` side-effect,
  serves `gateway.NewRouter` via `httptest`, asserts:
  - `GET /swagger/doc.json` → 200 + valid JSON + ≥ 2 entries in `paths`
  - `GET /swagger/index.html` → 200

- **`internal/gateway/config_test.go`** — `TestFromEnv` with 6 subtests covering all branches:
  defaults, custom GATEWAY_ADDR, custom VANTAGE_GATEWAY_MAX_ROWS, invalid non-integer, zero,
  and negative value (error paths). `FromEnv` coverage: 0% → 100%.

- **`internal/gateway/handler_test.go`** extended — `brokenPool` helper creates a non-nil
  `*pgxpool.Pool` pointing at localhost:19999 (lazy connect, fails on first query).
  `TestListGPUs_DBError_Unit` and `TestGetTelemetry_DBError_Unit` hit the 500 error paths.
  Combined: `internal/gateway` coverage 73.8% → 96.9%.

- **`Makefile`** — coverage target grep updated from `grep -v '/pkg/pb'` to
  `grep -v '/pkg/pb\|/pkg/docs'` excluding generated `pkg/docs` from the ≥90% gate.

- **`scripts/smoke/phase04-gateway.sh`** — executable `set -euo pipefail` smoke check:
  Postgres readiness probe, auto-seed one row if table empty, GPU discovery via `pg_exec`
  (no host psql), gateway startup + readiness wait (15s), 4 curl assertions (200 array,
  telemetry 200, 404 unknown GPU, swagger spec valid JSON with ≥ 2 paths), cleanup on EXIT.

- **`README.md`** — Phase 4 quickstart section: run gateway (env vars, `make build`), 3
  curl examples (list GPUs, telemetry, windowed telemetry), Swagger UI URL, `make swagger`
  regeneration note, `make smoke-04` description. Services + phase-status tables updated to ✅.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] swag CLI v1.16.4 generates docs.go with `LeftDelim`/`RightDelim` fields absent in `swag v1.8.1`**
- **Found during:** Task 1 (building cmd/gateway with the _ pkg/docs import)
- **Issue:** `go install github.com/swaggo/swag/cmd/swag@v1.16.6` installed the CLI at v1.16.4 (closest available release at time of install). The generated `pkg/docs/docs.go` uses `swag.Spec{LeftDelim: "{{", RightDelim: "}}"}` — fields added after v1.8.1. The existing indirect dep `github.com/swaggo/swag v1.8.1` did not compile.
- **Fix:** `go get github.com/swaggo/swag@v1.16.4` upgraded the runtime dependency. Build unblocked.
- **Files modified:** `go.mod`, `go.sum`
- **Commit:** 37d0d27

**2. [Rule 2 - Missing critical functionality] `internal/gateway/config.go:FromEnv` at 0% coverage — coverage gate at 87.3% (below 90%)**
- **Found during:** Task 2 (running `make coverage` for gate verification)
- **Issue:** `FromEnv` was introduced in Plan 04-01 but no tests covered it. With pkg/docs generated code now included, the module total coverage was 87.3%. The plan requires `make coverage` to pass.
- **Fix:** Added `internal/gateway/config_test.go` with `TestFromEnv` (6 subtests: defaults, custom addr, custom MaxRows, non-integer error, zero error, negative error). Also added `brokenPool` helper + DB-error-path tests to `handler_test.go` for ListGPUs and GetTelemetry error returns. Gateway: 73.8% → 96.9%; module: 87.3% → 90.4%.
- **Files modified:** `internal/gateway/config_test.go` (created), `internal/gateway/handler_test.go`
- **Commit:** 7d5d1f3
