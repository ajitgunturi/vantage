Add a check-protoc phony target to Makefile. Its recipe runs `command -v protoc` to detect the binary. If not found, it prints platform-specific install instructions (brew on macOS, apt on Debian/Ubuntu) and exits 1. If found, it runs `protoc --version` and exits 0. Create directory api/proto/ if it does not exist. Write api/proto/mq.proto with the following structure — use exact field names, numbers, and types as listed; field numbers are stable after initial commit and must never be renumbered:

File preamble: syntax proto3; package mq.v1; option go_package set to "github.com/ajitg/vantage/pkg/pb".

Message TelemetryMessage carries a single DCGM GPU metric reading. Fields at stable numbers: timestamp (string, field 1) — ISO 8601 UTC string; metric_name (string, 2) — e.g., DCGM_FI_DEV_GPU_UTIL; gpu_id (string, 3); device (string, 4) — e.g., nvidia0; uuid (string, 5) — GPU UUID; model_name (string, 6) — e.g., NVIDIA H100 80GB HBM3; hostname (string, 7); container (string, 8); pod (string, 9); namespace (string, 10); value (double, 11) — numeric metric value; labels_raw (string, 12) — raw Prometheus label string. Use string for field 1 (timestamp) rather than google.protobuf.Timestamp to avoid the timestamppb import in Phase 1; the Streamer restamps using time.RFC3339 in a later phase.

Message ProduceRequest: single field TelemetryMessage message at field number 1.

Message ProduceResponse: single field bool accepted at field number 1. Semantics: true means message was enqueued (drop-oldest may have fired silently); false only on validation error. Never used for backpressure — Produce is non-blocking by design.

Message ConsumeRequest: single field string consumer_id at field number 1. Marked optional in spirit (proto3 all fields are optional by default); used only for server-side stream correlation logging.

Service MQService with two RPCs:
  rpc Produce(ProduceRequest) returns (ProduceResponse) — unary; satisfies MQ-01 proto surface
  rpc Consume(ConsumeRequest) returns (stream TelemetryMessage) — server-streaming; satisfies MQ-02 proto surface

Create directory pkg/pb/ before running codegen (some protoc versions require the output directory to pre-exist). Then run `make proto`. The Makefile proto target invokes protoc with -I api/proto, --go_out=pkg/pb, --go_opt=paths=source_relative, --go-grpc_out=pkg/pb, --go-grpc_opt=paths=source_relative, and the glob api/proto/*.proto. The paths=source_relative option writes output directly into pkg/pb/ as mq.pb.go and mq_grpc.pb.go — it does NOT create a pkg/pb/mq.v1/ subdirectory.

Do not write or edit any file under pkg/pb/ manually. The generated files are always produced by make proto and are overwritten on each run.
