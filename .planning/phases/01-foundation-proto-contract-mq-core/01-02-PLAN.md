<objective>
Implement the Store interface and the in-memory bounded ring-buffer backend in the internal/queue package. This
is the MQ's core storage primitive: a pre-allocated fixed-capacity circular FIFO guarded by a single sync.Mutex,
with drop-oldest semantics when full. No channel, goroutine, or gRPC type appears in this package — it is a
pure storage seam that lets Phase 6's WAL backend slot in behind the same interface without touching consumers.

Purpose: Establish the thread-safe, race-verified storage layer that plan 01-03 (gRPC server + dispatch
goroutine) and plan 01-04 (HTTP inspect handler) depend on. The Store interface is MQ-08's WAL seam.

Output: internal/queue/store.go (interface + stats type), internal/queue/ring_store.go (RingStore), and
internal/queue/ring_store_test.go (unit tests + QA-02 concurrent correctness test).

Requirements covered: MQ-03 (each TryDequeue call returns at most one unique message — no fan-out at the store
layer), MQ-04 (sync.Mutex-guarded, no third-party primitives), MQ-05 (drop-oldest, no blocking), MQ-08 (Store
interface seam), QA-02 (concurrent delivery correctness under race detector).
</objective>

  <behavior>
    - Enqueue(nil) is not validated inside RingStore; precondition is that callers (MQServer.Produce) validate msg != nil before calling. RingStore stores pointers as-is.
    - Enqueue into an empty RingStore(3): Inspect().Depth == 1, Inspect().Dropped == 0, returns false.
    - Enqueue into a full RingStore(3) (already holding 3 items): head advances by 1, dropped counter increments by 1, new item written at tail, count stays at 3, returns true.
    - TryDequeue on empty RingStore returns (nil, false) without blocking.
    - TryDequeue on non-empty RingStore returns (msg, true) where msg is the oldest item, nil-clears the slot at the old head, advances head, decrements count.
    - Inspect() returns a StoreStats value copy: Depth == current count, Capacity == constructor cap argument, Dropped == cumulative drop events.
    - Close() returns nil (no-op for in-memory backend).
    - Compile-time assertion: var _ Store = (*RingStore)(nil) in ring_store.go ensures the compiler verifies RingStore satisfies Store.
    - NewRingStore(0) or NewRingStore(-1) panics with a descriptive message; callers must supply a positive capacity.
  </behavior>

  <action>
Create the directory internal/queue/ and two files inside it.

FILE 1 — internal/queue/store.go, package queue:

Import "github.com/ajitg/vantage/pkg/pb". Define StoreStats as a plain struct (safe to copy) with three exported fields: Depth int (messages currently buffered), Capacity int (total ring capacity; -1 signals unbounded — reserved for WAL backend), and Dropped int64 (cumulative messages overwritten by drop-oldest since process start).

Define the Store interface with exactly four methods. Enqueue accepts a *pb.TelemetryMessage and returns a bool (true when drop-oldest fired, false when a free slot was used). The method must never block and must never return an error — the in-memory backend consumes messages unconditionally. TryDequeue returns (*pb.TelemetryMessage, bool); the bool is false when the buffer is empty (non-blocking, returns immediately). Inspect returns a StoreStats value (not a pointer — callers receive a snapshot copy). Close returns error to satisfy io.Closer-style conventions; the in-memory backend returns nil.

Add a doc comment above Store explaining the WAL seam: a WAL-backed implementation satisfying this interface will be added in Phase 6 without modifying any consumer code. Also note the contract for Enqueue: drop-oldest fires when at capacity; Produce therefore never blocks.

FILE 2 — internal/queue/ring_store.go, package queue:

Import "sync" and "github.com/ajitg/vantage/pkg/pb".

Define RingStore as a struct with unexported fields: mu sync.Mutex (single lock guards all state — do not use sync.RWMutex, every TryDequeue mutates head/count making RWMutex semantically incorrect here per RESEARCH.md), buf []*pb.TelemetryMessage (pre-allocated in NewRingStore; never grows or shrinks after construction), head int (index of the oldest item), tail int (index where the next Enqueue writes), count int (number of valid items currently in the buffer, invariant: 0 <= count <= cap), cap int (maximum capacity, set once at construction), dropped int64 (guarded by mu; not atomic — the lock is always held when this field is read or written).

Implement NewRingStore(capacity int) *RingStore: if capacity <= 0, panic with "queue: RingStore capacity must be positive, got <value>"; allocate buf as make([]*pb.TelemetryMessage, capacity) with capacity as both length and cap; set the cap field to the capacity argument; return the pointer.

Implement Enqueue(msg *pb.TelemetryMessage) bool: acquire mu (defer mu.Unlock). When count equals cap (buffer full), execute drop-oldest: nil-clear buf[head] to release the pointer for GC, advance head as (head+1)%cap, decrement count by 1, increment dropped by 1, set a local dropped-flag to true. Then unconditionally write msg to buf[tail], advance tail as (tail+1)%cap, increment count by 1. Return the dropped-flag (true only on overflow, false on normal insert). Critical lock discipline: mu.Unlock() must happen via defer before any channel send — but Enqueue itself does not touch any channel, so defer is safe here.

Implement TryDequeue() (*pb.TelemetryMessage, bool): acquire mu (defer mu.Unlock). If count == 0, return nil, false immediately (the deferred unlock fires). Copy buf[head] into a local variable msg, nil-clear buf[head] (releases GC reference), advance head as (head+1)%cap, decrement count by 1. Return msg, true. The lock is released by defer before the caller can use the returned pointer — no mutex escapes this method.

Implement Inspect() StoreStats: acquire mu (defer mu.Unlock). Return a StoreStats value with Depth set to count, Capacity set to cap, Dropped set to dropped. Returning a value (not pointer) means the caller has a snapshot that cannot race with future mutations.

Implement Close() error: return nil.

Add the compile-time interface satisfaction assertion as a package-level blank-identifier variable declaration: var _ Store = (*RingStore)(nil). Place it at the top of ring_store.go after the import block so a compile failure is immediately visible.
