## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] go mod tidy removed testify before proto code existed**

- **Found during:** Task 1 execution
- **Issue:** `go mod tidy` (run after `go get`) removed all three deps because no Go source files existed yet to import them. grpc and protobuf were re-added by `make proto` (which generates pkg/pb that imports them), but testify had no importer.
- **Fix:** Deferred `go mod tidy` until after `make proto` succeeded (so grpc/protobuf are kept via pkg/pb imports). Added testify back via `go get github.com/stretchr/testify@v1.11.1` without a subsequent tidy. Testify will be properly anchored as a direct test dependency once `_test.go` files are created in plan 01-02.
- **Files modified:** go.mod, go.sum
- **Impact:** None on correctness; go mod verify passes; all three packages are in go.mod as required.

**2. [Rule 3 - Blocking] protoc-gen-go and protoc-gen-go-grpc plugins not on PATH**

- **Found during:** Task 2 execution (pre-check before running make proto)
- **Issue:** Go installs binaries to `$(go env GOPATH)/bin` which was not on the shell PATH in the agent environment.
- **Fix:** Prepended `$(go env GOPATH)/bin` to PATH when invoking `make proto`. The generated output is identical; this is a shell environment issue, not a code issue.
- **Files modified:** None — make proto ran successfully.
