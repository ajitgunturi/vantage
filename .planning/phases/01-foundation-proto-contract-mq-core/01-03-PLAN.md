## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| gRPC client → Produce/Consume RPCs | Untrusted proto bytes over cluster-internal TCP; can be malformed (nil message, oversized fields) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-03-01 | Tampering | Produce RPC — nil message field | high | mitigate | Guard req.GetMessage()==nil before store.Enqueue; return codes.InvalidArgument; a nil pointer reaching ring buffer Enqueue would panic and crash the MQ process |
| T-01-03-02 | DoS | Consume RPC — goroutine exhaustion via many open streams | medium | accept | activeC counter provides real-time visibility via inspect; grpc-go default MaxConcurrentStreams=100 per HTTP/2 connection limits exposure; explicit cap can be added via grpc.MaxConcurrentStreams in Phase 5 |
| T-01-03-03 | Information Disclosure | GET /api/v1/queue/inspect | low | accept | Exposes queue depth, throughput, and consumer count; cluster-internal port only; not reachable via Ingress or NodePort in Phase 1; document no-external-exposure requirement in Phase 5 Helm values |
| T-01-03-04 | DoS | labels_raw string field — unbounded length | low | accept | grpc-go enforces 4MB default max receive message size at the HTTP/2 framing layer before Produce handler is invoked; no explicit field-level check required for telemetry payload sizes |
