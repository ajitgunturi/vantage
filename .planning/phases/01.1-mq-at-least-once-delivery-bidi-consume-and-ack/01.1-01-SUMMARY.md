# Phase 01.1 Plan 01: Proto Contract Upgrade Summary

**One-liner:** bidi Consume RPC with per-message broker id (uint64 field 13) and ConsumeClientMsg (credit + ack_id) added to mq.proto; pkg/pb stubs regenerated.

## What Was Built

Upgraded `api/proto/mq.proto` to implement the broker-side at-least-once delivery contract (ADR-001 / D-03 / D-06):

1. **`TelemetryMessage.id` (field 13)** — broker-assigned monotonic `uint64` delivery id. Zero from Streamer (ignored); overwritten by MQServer on dequeue. Field 13 preserves 1-byte varint encoding (fields 1-15 use 1-byte tags).

2. **`ConsumeClientMsg` message** — new client→server message for the bidi stream:
   - `ack_id uint64 = 1` — broker id being acknowledged (0 on initial credit message)
   - `credit int32 = 2` — initial in-flight window size (meaningful only on first message)
   - `consumer_id string = 3` — consumer instance identifier (logging only; truncated to 64 bytes per security constraint)

3. **Bidi `Consume` RPC** — changed from `rpc Consume(ConsumeRequest) returns (stream TelemetryMessage)` to `rpc Consume(stream ConsumeClientMsg) returns (stream TelemetryMessage)`. The generated server interface becomes `Consume(grpc.BidiStreamingServer[ConsumeClientMsg, TelemetryMessage]) error`; client becomes `Consume(ctx) (MQService_ConsumeClient, error)`.

4. **`ConsumeRequest` retained as deprecated** — marked with comment noting it is superseded by `ConsumeClientMsg` per ADR-001; will be removed in a future cleanup phase.

`pkg/pb` stubs were regenerated via `PATH="$(go env GOPATH)/bin:$PATH" make proto` (protoc-gen-go + protoc-gen-go-grpc installed at `/Users/ajitg/go/bin/`).
