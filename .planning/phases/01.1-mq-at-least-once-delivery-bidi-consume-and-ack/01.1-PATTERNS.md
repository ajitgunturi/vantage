### `internal/queue/ring_store.go` (store impl, extension)

**Analog:** `internal/queue/ring_store.go` (self)

**Compile-time assertion pattern** (line 12) ‚Äî copy exactly for the updated interface:
```go
var _ Store = (*RingStore)(nil)
```
This will fail at compile time if `Requeue` is added to the interface but not implemented on `RingStore`.

**Mutex pattern to replicate** (lines 54‚Äì71 for `Enqueue`, lines 76‚Äì88 for `TryDequeue`):
- Acquire `r.mu.Lock()` at top of method, `defer r.mu.Unlock()`
- All mutations under the same mutex guard; no channels called while the lock is held
- Nil out released slots: `r.buf[r.head] = nil` before advancing head pointer (GC friendliness)
- Modular arithmetic: `r.head = (r.head + 1) % r.cap`

**Ring buffer field layout** (lines 24‚Äì32):
```go
type RingStore struct {
    mu      sync.Mutex
    buf     []*pb.TelemetryMessage
    head    int    // oldest valid item
    tail    int    // next Enqueue writes here
    count   int    // invariant: 0 <= count <= cap
    cap     int
    dropped int64  // guarded by mu
}
```

**`Requeue` must use front-insertion** (decrement `head` in reverse-iteration order so first element ends up at head after all insertions):
```go
// Insert msgs in reverse so msgs[0] ends up at head.
for i := len(msgs) - 1; i >= 0; i-- {
    if r.count == r.cap {
        // Drop newest (tail-1) to prioritize redelivery
        r.tail = (r.tail - 1 + r.cap) % r.cap
        r.buf[r.tail] = nil
        r.count--
        r.dropped++
    }
    r.head = (r.head - 1 + r.cap) % r.cap
    r.buf[r.head] = msgs[i]
    r.count++
}
```

## Shared Patterns

### Lock Discipline (applies to all new code in `server.go` and `ring_store.go`)

**Source:** `internal/server/server.go` lines 1‚Äì9 (package comment), `internal/queue/ring_store.go` lines 54‚Äì88

**Rule:** Channel sends (`sem <- struct{}{}`, `ackCh <- id`, `close(notifyCh)`) MUST NOT be called while any mutex is held. The store releases its mutex via `defer r.mu.Unlock()` before returning from `TryDequeue`/`Enqueue`/`Requeue`. After those calls return, the caller has no lock and may safely send to channels.

### Atomic Counter Pattern (all new counters in `server.go`)

**Source:** `internal/server/server.go` lines 45‚Äì47, 148

All counter increments use `atomic.AddInt64`; all reads in `Stats()` use `atomic.LoadInt64`. No mutex around individual counter operations.

### Compile-Time Interface Assertion

**Source:** `internal/queue/ring_store.go` line 12

```go
var _ Store = (*RingStore)(nil)
```

After adding `Requeue` to the `Store` interface, this line will refuse to compile until `RingStore.Requeue` is implemented. Place equivalent assertions in test files for mock types:

```go

```

### gRPC Error Handling Pattern

**Source:** `internal/server/server.go` line 114

Use `status.Error(codes.InvalidArgument, "...")` for validation errors at the gRPC boundary. Do not return raw Go errors from RPC handlers.

| Old Code | New Code | Reason |
|----------|----------|--------|
| `dispatch()` goroutine | Eliminated | Root of at-most-once bug; per-consumer poll replaces it |
| `workCh chan *pb.TelemetryMessage` | Eliminated | In-flight buffer that caused silent loss on disconnect |
| `notify chan struct{}` (buffered 1) | `notifyCh chan struct{}` (close-and-reopen) | Old channel woke only one consumer; close wakes all |
| `Consume(req *pb.ConsumeRequest, stream pb.MQService_ConsumeServer)` | `Consume(stream pb.MQService_ConsumeServer)` | Bidi RPC signature has no request argument |
