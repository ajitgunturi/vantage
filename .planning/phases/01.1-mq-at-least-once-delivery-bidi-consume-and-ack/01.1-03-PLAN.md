<objective>
Replace the at-most-once dispatch engine with the bidirectional, credit-gated, lease-tracked
at-least-once engine (MQ-09 + MQ-10, ADR-001). This is the heart of the phase: remove the
shared `workCh` + `dispatch()` goroutine (the root of the reproduced silent-loss defect),
rewrite `Consume` as a two-goroutine bidi handler (one recv loop, one send loop) with a credit
semaphore and a per-consumer lease table, re-enqueue unacked leases on disconnect, and add the
`delivered`/`consumed`(=acks)/`redelivered`/`in_flight` counters.

Tests are the spec (D-10): the bidi mock and the five required `-race` scenarios are written
alongside the engine in this plan because, in Go, the tests can only compile against the new
`NewMQServer`/`Stats`/`Consume` API — engine and tests are one atomic TDD unit.

Purpose: deliver broker-side at-least-once on the live path with a bounded in-flight window.
Output: rewritten `internal/server` engine, bidi test harness + 5 D-10 race tests, rewired `cmd/mq`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Collector(client) → Consume bidi stream | Client controls initial credit, ack ids, consumer_id, and connect/disconnect timing — all untrusted. |
| recv goroutine → send loop | Internal channel boundary; lease table is single-owner (send loop) to avoid shared-state races. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01.1-01 | Tampering | ack handling (`leases[id]`) | high | mitigate | Before delete+replenish, require `id` present in THIS consumer's lease table; unknown/double acks are no-ops — prevents acking another consumer's lease and credit inflation. Proven by `TestMQ_AckSafety` + `TestMQ_NoOverPull`. |
| T-01.1-02 | Denial of Service | per-consumer lease table growth | medium | accept | Lease table is bounded by credit C (no over-pull); a connected never-acking consumer simply stalls its own window. No lease/visibility timeout this phase (D-04, deferred) — accepted residual, documented in ADR-001. |
| T-01.1-03 | Denial of Service | initial credit value | high | mitigate | Clamp on the first ConsumeClientMsg: `credit <= 0 -> defaultCredit`; apply a ceiling (`max(BufferSize,1000)`) to bound the semaphore allocation — blocks negative/zero/huge credit abuse. |
| T-01.1-04 | Denial of Service | redelivery on rapid churn | medium | mitigate | Requeue is bounded by ring capacity (drop-tail when full, Plan 02); redelivered counter is observable via inspect. |
| T-01.1-05 | Denial of Service (resource exhaustion) | recv goroutine on disconnect | high | mitigate | `defer wg.Wait()` joins the recv goroutine before the handler returns (Pitfall 1); `TestMQ_GoroutineLeak` enforces baseline goroutine count under `-count=50`. |
| T-01.1-06 | Tampering | `consumer_id` string | low | mitigate | Used for logging only; do not forward downstream; truncate to first 64 bytes if logged. |
</threat_model>
