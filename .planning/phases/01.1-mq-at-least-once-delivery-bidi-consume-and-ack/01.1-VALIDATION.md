# Phase 01.1 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.
> The bidi at-least-once rewrite is concurrency-critical — tests ARE the spec (CONTEXT D-10).

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | `go test` (stdlib testing + testify; `-race` mandatory for MQ) |
| **Config file** | none — Go toolchain; Makefile drives gates |
| **Quick run command** | `go test -race -count=1 ./internal/server/ ./internal/queue/` |
| **Full suite command** | `make test` ( `go test -race -covermode=atomic ./...` ) then `make coverage` (≥90%) |
| **Estimated runtime** | ~5–15 s quick; ~30–60 s full with `-race` |

> Phase-1 exit discipline carries forward: the MQ concurrency tests must survive `go test -race -count=50` (no flakes, no deadlocks) before the phase is done.

---

## Sampling Rate

- **After every task commit:** Run the quick command for the touched package(s)
- **After every plan wave:** Run `make test` + `make coverage`
- **Before `/gsd-verify-work`:** Full suite green, `-race -count=50` clean, coverage ≥90%
- **Max feedback latency:** ~15 seconds (quick), ~60 seconds (full)
