# Phase 01.1 Plan 03: Bidi At-Least-Once Engine Summary

**One-liner:** Replaced the at-most-once `workCh`+`dispatch` engine (the silent-loss defect root) with a bidirectional, credit-gated, lease-tracked at-least-once engine — a message leaves broker custody only on ack of its broker-assigned id, unacked leases are requeued oldest-first and redelivered to a survivor on disconnect, and in-flight is structurally bounded by a credit semaphore. Proven under `go test -race -count=50`.

## Deviations from Plan

1. **Executed inline by the orchestrator, not a gsd-executor subagent.** Two subagent dispatches stalled on the stream watchdog (600s, no progress) before making any changes. With the user's explicit confirmation, the orchestrator executed the plan inline on the main tree (worktrees were already auto-degraded to sequential per #683). No partial state was left by the stalled attempts.
2. **Park-select ack bug fix (engine correctness, not in the plan).** RESEARCH Pattern 1's empty-store park used `case <-ackCh: // outer loop will drain it`, which discards the ack (a channel receive consumes the value) — stalling `Consumed`. Fixed to process the ack inline (delete lease, `consumed++`, replenish token). Documented in decisions.
3. **`Shutdown()` no longer stops live consumers.** The new model stops consumers on stream-context cancellation (gRPC GracefulStop), so the old `TestMQServer_Shutdown_ClosesConsume` was reframed as `TestMQServer_Shutdown_ReturnsOnDisconnect` (cancel ctx → Consume returns cleanly; Shutdown idempotent).

## Threat Flags

Engine threat mitigations applied inline (per the plan's STRIDE register):

- **T-01.1-01 (ack tampering):** ack(id) is a no-op unless `id` is a live lease held by THIS consumer — unknown/double acks neither delete a lease nor replenish credit. Proven by `TestMQ_AckSafety` + `TestMQ_NoOverPull`.
- **T-01.1-03 (initial-credit DoS):** initial credit clamped `<=0 -> defaultCredit`, ceiling `max(Capacity,1000)` bounds the semaphore allocation.
- **T-01.1-05 (recv-goroutine leak):** `defer wg.Wait()` joins the recv goroutine before the handler returns; `TestMQ_GoroutineLeak` enforces baseline under `-count=50`.
- **T-01.1-02 / T-01.1-04 (lease growth / redelivery churn):** lease table bounded by credit C; Requeue bounded by ring capacity (drop-newest, Plan 02). Accepted residual per ADR-001 (no lease/visibility timeout this phase).
