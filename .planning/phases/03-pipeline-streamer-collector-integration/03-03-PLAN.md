## Phase Goal

**As a** platform operator, **I want to** stream live DCGM GPU telemetry from the CSV through the message queue into PostgreSQL, **so that** every reading is durably stored exactly once even when producers and collectors run concurrently.

This plan delivers the MQ -> PostgreSQL slice (the consumer half).

## Architecture note (read before coding)

Service logic lives in `internal/collector` (package `collector`), mirroring `cmd/mq` -> `internal/server`. `cmd/collector/main.go` is a thin `package main` wrapper that also wires `pkg/db` (FromEnv -> Migrate -> New -> pool). This keeps logic in the `./internal/...` coverage-gated scope and lets the QA-03 E2E test (plan 03-04) import the exported `collector.Consume` seam. Integration tests run under `-tags=integration` (included by `make coverage`) and require the Rancher Docker env.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MQ -> Collector | Untrusted/duplicated TelemetryMessage payloads arrive over the bidi stream |
| Collector -> PostgreSQL | Field values become SQL parameters; connection string is a secret |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-03a | Tampering | persistBatch SQL | high | mitigate | Parameterized args ($1..$11) via pgx.Batch.Queue + models.InsertSQL; never string-interpolate CSV/proto field content into SQL — closes SQL-injection from malicious field values |
| T-03-03b | DoS (self) | Run reconnect loop | medium | mitigate | Exponential backoff base 100ms, max 5s cap — prevents CPU spin / thundering herd on a permanently-down MQ |
| T-03-03c | Information Disclosure | error strings / logs | medium | mitigate | Errors prefixed `collector:` and free of the DSN value (reuse pkg/db redaction convention); skip-and-log bad proto without dumping payloads |
| T-03-03d | DoS | unbounded in-flight / pool exhaustion | low | mitigate | Credit window bounds outstanding-unacked; pgx.BatchResults drained exactly b.Len() times with defer Close() so pool conns are never leaked |
| T-03-SC | Tampering | dependency installs | low | accept | Zero new packages; uses existing pgx/v5, grpc, testcontainers, pkg/db, pkg/models |

T-03-03a is rated **high** (SQL injection class) and is fully mitigated by parameterized queries — no string interpolation of field content anywhere in the Collector.
</threat_model>
