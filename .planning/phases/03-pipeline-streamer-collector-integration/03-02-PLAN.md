## Phase Goal

**As a** platform operator, **I want to** stream live DCGM GPU telemetry from the CSV through the message queue into PostgreSQL, **so that** every reading is durably stored exactly once even when producers and collectors run concurrently.

This plan delivers the CSV -> MQ slice (the producer half).

## Architecture note (read before coding)

Service logic lives in `internal/streamer` (package `streamer`), mirroring `cmd/mq` -> `internal/server`. `cmd/streamer/main.go` is a thin `package main` wrapper. This places the logic in the `./internal/...` coverage-gated scope (the Makefile `coverage` target lists `./internal/... ./pkg/...`) and lets the QA-03 E2E test (plan 03-04) import the exported `streamer.Stream` seam. CLAUDE.md sanctions per-service `internal` helpers ("a service in cmd/<svc> may import ... its own internal helpers").
