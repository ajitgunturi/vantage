## Phase Goal

**As a** platform operator, **I want to** stream live DCGM GPU telemetry from the CSV through the message queue into PostgreSQL, **so that** every reading is durably stored exactly once even when producers and collectors run concurrently.

This plan is the end-to-end proof of the full slice.

## Architecture note (read before coding)

The E2E test imports the exported seams `streamer.Stream` (publisher, single pass) and `collector.Consume` (one consumer stream) from the `internal/` packages — possible because Phase 3 placed service logic in `internal/streamer` and `internal/collector` rather than `package main`. The test lives under `test/e2e/` which is intentionally outside the `./internal/... ./pkg/...` coverage-gated scope (it is a correctness proof, not a coverage contributor; per-service integration tests in plans 02 and 03 already drive `internal/*` coverage). `make smoke-03` is served by the existing Makefile `smoke-%` pattern rule, so no Makefile change is needed.
