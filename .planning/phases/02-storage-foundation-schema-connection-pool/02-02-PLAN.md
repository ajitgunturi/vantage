<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env → cmd/migrate | `VANTAGE_DB_DSN` crosses from environment into the one-shot migrate runner |
| dev-stack creds → repo | docker-compose dev credentials are committed (local-dev only) |
| smoke SQL → DB | seed + EXPLAIN SQL issued by the smoke script crosses into Postgres |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-06 | Information Disclosure | docker-compose committed credentials | medium | mitigate | Compose creds are inert LOCAL-DEV defaults (vantage/vantage on localhost), annotated as such; production DSN is env-injected via VANTAGE_DB_DSN in Phase 5, never from compose |
| T-02-07 | Information Disclosure | cmd/migrate error/log output | high | mitigate | cmd/migrate logs success/failure context only; never logs the DSN value (reuses pkg/db's redaction discipline from plan 02-01) |
| T-02-08 | Tampering | seed/EXPLAIN SQL in smoke script | low | mitigate | Smoke SQL uses fixed literals / generate_series with no untrusted external input; no user-supplied values concatenated |
| T-02-09 | Denial of Service | smoke script leaves dev stack / volume running | low | accept | Intentional — stack stays up for inspection; `make dev-down` tears it down on demand; named volume is local-only |
</threat_model>
